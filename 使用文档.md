## 快速开始

### 1. 添加依赖

```xml
<dependency>
    <groupId>zmxe.cn</groupId>
    <artifactId>mybatis-plus-stream-boot-starter</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

### 2. 定义实体类

```java
@Data
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String username;
    private String nickname;
    private String role;
    private Integer creditScore;
    private LocalDateTime createdAt;
}
```

### 3. 定义 Mapper

```java
public interface UserMapper extends MysqlBaseMapper<User> {
}
```

### 4. 定义 Service 接口

```java
public interface UserService extends IMysqlServiceBase<User> {
}
```

### 5. 定义 Service 实现

```java
@Service
public class UserServiceImpl extends MysqlServiceBaseImpl<UserMapper, User> implements UserService {
}
```

完成以上步骤后，`UserService` 即拥有本框架提供的所有扩展方法。

## 基础查询 - get

### get(SFunction, Object) - 按列值查询单条

```java
// 按单列查询
User user = userService.get(User::getUsername, "user1");

// 值为 null 时自动转为 IS NULL 查询
User user = userService.get(User::getOpenid, null);

// 值为 Collection 时自动转为 IN 查询，返回第一条
User user = userService.get(User::getId, List.of(1L, 2L, 3L));
```

### get(Consumer) - 按条件查询单条

```java
User user = userService.get(w -> w.eq(User::getUsername, "user1"));

// 组合条件
User user = userService.get(w -> w
    .eq(User::getRole, "admin")
    .eq(User::getUsername, "admin")
);
```

### get(Consumer, Consumer, Class) - 自定义 Select 查询

```java
// 查询并映射到 DTO
UserDTO dto = userService.get(
    w -> w.eq(User::getId, 1L),
    s -> s.select(User::getUsername).select(User::getNickname),
    UserDTO.class
);
```

### getOrDefault - 带默认值的查询

```java
User defaultUser = new User();
defaultUser.setUsername("guest");

// 查不到时返回默认值
User user = userService.getOrDefault(User::getUsername, "nonexistent", defaultUser);
// user.getUsername() == "guest"

// 条件版本
User user = userService.getOrDefault(
    w -> w.eq(User::getRole, "superadmin"),
    defaultUser
);
```

## 获取单值 - getValue

### getValue(SFunction, Object, SFunction) - 按列查询单个字段值

```java
// 查询 user1 的昵称
String nickname = userService.getValue(User::getUsername, "user1", User::getNickname);
```

### getValue(Consumer, SFunction) - 按条件查询单个字段值

```java
String nickname = userService.getValue(
    w -> w.eq(User::getId, 1L),
    User::getNickname
);
```

### getValue(Consumer, Function) - 使用聚合函数获取值

```java
// 获取最大 ID
Function<GroupFunctionLambdaQueryWrapper, Object> maxFunc = x -> x.max(User::getId);
Object maxId = userService.getValue(w -> {}, maxFunc);

// 获取总数
Function<GroupFunctionLambdaQueryWrapper, Object> cntFunc = x -> x.count(User::getId);
Object count = userService.getValue(w -> {}, cntFunc);

// 获取信用分总和
Function<GroupFunctionLambdaQueryWrapper, Object> sumFunc = x -> x.sum(User::getCreditScore);
Object totalCredit = userService.getValue(w -> {}, sumFunc);
```

## 列表查询 - list

### list(SFunction, Object) - 按列值查询列表

```java
// 查询所有 role=user 的用户
List<User> users = userService.list(User::getRole, "user");

// 按 Collection 查询（IN 查询）
List<User> users = userService.list(User::getId, List.of(1L, 2L, 3L));
```

### list(Consumer) - 按条件查询列表

```java
List<User> users = userService.list(w -> w.eq(User::getRole, "user"));

// 组合条件
List<User> users = userService.list(w -> w
    .eq(User::getRole, "user")
    .gt(User::getCreditScore, 80)
);
```

### list(Consumer, Consumer, Integer) - 排序 + 限制条数

```java
// 按 ID 倒序查询前 3 条
List<User> users = userService.list(
    w -> {},                           // 无过滤条件
    o -> o.orderDesc(User::getId),     // 排序
    3                                   // 限制条数
);
```

### list(Consumer, Consumer, Integer, Consumer, Class) - 自定义 Select + 排序 + DTO 映射

```java
List<UserDTO> dtos = userService.list(
    w -> w.eq(User::getRole, "user"),                             // 过滤
    o -> o.orderAsc(User::getCreatedAt),                          // 排序
    10,                                                            // 限制
    s -> s.select(User::getId).select(User::getUsername),          // 选择字段
    UserDTO.class                                                  // 映射类型
);
```

## 条件操作符

所有条件操作符均支持 `condition` 参数（布尔值，为 false 时跳过该条件），以下示例省略 `condition` 参数。

### 比较操作符

```java
// 等于 eq
w -> w.eq(User::getRole, "user")

// 不等于 ne
w -> w.ne(User::getRole, "admin")

// 大于 gt
w -> w.gt(User::getCreditScore, 80)

// 大于等于 ge
w -> w.ge(User::getCreditScore, 100)

// 小于 lt
w -> w.lt(User::getId, 10L)

// 小于等于 le
w -> w.le(User::getId, 10L)
```

### 条件动态拼接

```java
// condition 参数：为 false 时跳过该条件，非常适合动态查询
String role = request.getRole();       // 可能为 null
String keyword = request.getKeyword(); // 可能为 null

List<User> users = userService.list(w -> w
    .eq(role != null, User::getRole, role)
    .likeDefault(keyword != null, User::getUsername, keyword)
);
```

### 范围操作符

```java
// BETWEEN
w -> w.between(User::getId, 2L, 5L)

// NOT BETWEEN
w -> w.notBetween(User::getId, 2L, 5L)
```

### IN / NOT IN

```java
// IN (集合)
w -> w.in(User::getId, List.of(1L, 2L, 3L))

// IN (可变参数)
w -> w.in(User::getId, 1L, 2L, 3L)

// NOT IN
w -> w.notIn(User::getId, List.of(999L))
```

### IN 子查询

```java
// IN (子查询)
w -> w.inSubSql(User::getId, sub -> sub
    .from(Order.class)
    .select(Order::getUserId)
    .filter(sw -> sw.eq(Order::getStatus, "completed"))
)
```

### 空值判断

```java
// IS NULL
w -> w.isNull(User::getOpenid)

// IS NOT NULL
w -> w.isNotNull(User::getUsername)
```

### 模糊匹配

```java
// LIKE (不带通配符，需自行添加 %)
w -> w.like(User::getUsername, "%user%")

// likeDefault: LIKE '%val%' (自动添加两侧通配符)
w -> w.likeDefault(User::getUsername, "user")

// likeLeft: LIKE '%val' (左通配)
w -> w.likeLeft(User::getUsername, "1")

// likeRight: LIKE 'val%' (右通配)
w -> w.likeRight(User::getUsername, "user")

// NOT LIKE
w -> w.notLike(User::getUsername, "%test%")
```

### 正则匹配

```java
// REGEXP
w -> w.regexp(User::getUsername, "^user[0-9]+$")
```

### 全文搜索

```java
// MATCH ... AGAINST
w -> w.match("搜索词", User::getTitle, User::getContent)

// MATCH IN BOOLEAN MODE
w -> w.matchInBooleanMode("+必须 -排除", User::getTitle, User::getContent)
```

### 列与列比较

```java
// 比较两列
w -> w.eqColumn(User::getCreatedAt, User::getUpdatedAt)
```

## 逻辑组合 - and / or / not

```java
// AND 嵌套 (WHERE role='user' AND (credit > 80))
w -> w.eq(User::getRole, "user")
      .and(sub -> sub.gt(User::getCreditScore, 80))

// OR 嵌套 (WHERE role='admin' OR (role='user'))
w -> w.eq(User::getRole, "admin")
      .or(sub -> sub.eq(User::getRole, "user"))

// 简单 OR（下一个条件用 OR 连接）
w -> w.eq(User::getRole, "admin")
      .or()
      .eq(User::getRole, "user")

// NOT 嵌套
w -> w.not(sub -> sub.eq(User::getRole, "banned"))
```

## 聚合函数

在 `getValue`、`listValues`、`listGroupValues` 以及 `selectFunc` 中可使用聚合函数：

```java
// COUNT(*)
x -> x.count()

// COUNT(column)
x -> x.count(User::getId)

// COUNT(DISTINCT column)
x -> x.countDistinct(User::getRole)

// SUM(column) - 返回 BigDecimal
x -> x.sum(User::getCreditScore)

// SUM(column) 默认为 0（IFNULL(SUM(...), 0)）
x -> x.sumDefaultZero(User::getCreditScore)

// AVG(column) - 返回 BigDecimal
x -> x.avg(User::getCreditScore)

// MIN(column)
x -> x.min(User::getId)

// MAX(column)
x -> x.max(User::getId)

// GROUP_CONCAT(column)
x -> x.groupConcat(User::getUsername)

// GROUP_CONCAT(column SEPARATOR ',')
x -> x.groupConcat(User::getUsername, ",")

// GROUP_CONCAT(DISTINCT column)
x -> x.groupConcatDistinct(User::getUsername)
```

### 条件聚合

```java
// COUNT(CASE WHEN ... THEN 1 END) - 条件计数
x -> x.countPredicate(w -> w.eq(User::getRole, "admin"))

// 条件去重计数
x -> x.countPredicateDistinct(w -> w.gt(User::getCreditScore, 80), User::getId)
```

## Stream 流式 API

框架提供底层 Stream API，可直接使用进行更灵活的查询：

```java
// 获取查询流
MybatisQueryableStream1<User, User> stream = userService.stream();

// 流式查询示例
List<User> users = userService.stream()
    .filter(w -> w.eq(User::getRole, "user"))
    .sorted(o -> o.orderDesc(User::getId))
    .limit(10)
    .map(s -> s.selectAll(User.class), User.class)
    .collect(Collectors.toList());

// 流式查询第一条
Optional<User> first = userService.stream()
    .filter(w -> w.eq(User::getUsername, "user1"))
    .map(s -> s.selectAll(User.class), User.class)
    .findFirst();

// distinct 去重
List<User> users = userService.stream()
    .filter(w -> {})
    .distinct()
    .map(s -> s.selectAll(User.class), User.class)
    .collect(Collectors.toList());

// mapToValue 获取单值流
List<String> names = userService.stream()
    .filter(w -> w.eq(User::getRole, "user"))
    .mapToValue(x -> x.column(User::getUsername))
    .collect(Collectors.toList());
```

### 可执行流 - ExecutableStream

```java
// 获取可执行流
MybatisExecutableStream<User> execStream = userService.executableStream();

// 流式删除
int deleted = userService.executableStream()
    .filter(w -> w.eq(User::getRole, "banned"))
    .executeDelete();

// 流式更新
int updated = userService.executableStream()
    .filter(w -> w.lt(User::getCreditScore, 0))
    .set(s -> s.set(User::getCreditScore, 0))
    .executeUpdate();
```

### 测试过程中发现并修复的框架 Bug

| Bug | 根因 | 修复文件 |
|-----|------|---------|
| `get(SFunction, Object)` SQL 参数为 null | MBP 3.5.9 中 `WRAPPER_PARAM_MIDDLE` 格式变化导致参数 key 替换失效 | `LambdaQueryWrapper.java` |
| `saveBatchWithoutId` / `saveIgnore` / `saveReplace` 参数绑定失败 | Mapper 方法的 `columns` 和 `values` 参数缺少 `@Param` 注解 | `MysqlBaseMapper.java` |
| `groupConcat` 带 SEPARATOR SQL 语法错误 | separator 值未用单引号包裹 | `GroupFunctionLambdaQueryWrapper.java` |
| `set(condition=false)` 未跳过赋值 | `setFunc` 方法未检查 `condition` 参数 | `AbstractSetLambdaQueryWrapper.java` |
| `bXorFunc` 运算符错误 | 按位异或使用了 `~`（取反）而非 `^`（异或） | `AbstractFunctionLambdaQueryWrapper.java` |
